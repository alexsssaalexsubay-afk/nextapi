# 快速开始

**目标：10 分钟内生成第一条视频。**

本指南使用 **Batch Studio**——一款本地运行的桌面工具，不需要写代码。上传一张 CSV，点击按钮，视频自动下载到本地。

---

## 开始前的准备

你需要三样东西：

| 准备项 | 在哪里获取 |
|--------|-----------|
| API 密钥（`sk_live_…`） | [app.nextapi.top](https://app.nextapi.top) → **密钥 → 新建密钥** |
| Python 3.11 及以上 | [python.org/downloads](https://python.org/downloads) |
| 足够的余额 | 控制台 → **计费**（一条 5 秒镜头约消耗 1–2 积分） |

---

## 第一步：安装 Batch Studio

打开终端，依次执行以下命令：

```bash
cd toolkit/batch_studio
python -m venv .venv
source .venv/bin/activate       # Windows 用: .venv\Scripts\activate
pip install -r requirements.txt
streamlit run app.py
```

Streamlit 启动后会打印一个地址，例如 `http://localhost:8501`。在浏览器中打开即可。

:::tip 保持浏览器标签页开着
整个批量运行期间，浏览器标签页必须保持打开。关闭标签页会中断任务。
:::

---

## 第二步：填入 API 密钥

在左侧边栏找到 **连接设置** 区域：

- **接口地址** — 默认保持 `https://api.nextapi.top` 不要修改
- **API 密钥（sk_live_…）** — 把你的密钥粘贴在这里

密钥识别成功后，状态指示灯会变成**绿色**。

:::warning 密钥要保密
请把它当成密码对待。不要在截图、Slack、邮件里分享密钥。如果泄露，立即去控制台撤销并重新生成。
:::

---

## 第三步：上传示例清单

项目里附带了一份 15 条镜头的示例清单，路径是：

```
toolkit/batch_studio/sample_data/shot_manifest.csv
```

在 **Batch** 标签页，把这个文件拖拽上传（或点击选择文件）。上传后你会看到：

- 清单预览表格，共 15 行，每行一条镜头
- 统计栏：**15 条镜头 · 5 个连贯组 · 0 张参考图待上传**
- 按集数和场景分组的清单视图

这份示例覆盖了两集短剧和一组电商产品镜头，可以完整测试所有功能。

---

## 第四步：先跑快速测试（3 条镜头）

在提交完整批量任务前，点击 **⚡ 快速测试（3 条镜头）**。

这只会提交前三行，用来快速确认：

1. 你的 API 密钥有效且被接受
2. 接口地址可以正常访问
3. API 返回了正常的任务 ID 和视频链接
4. 视频成功下载到了本地输出目录

快速测试通常 **2–5 分钟**完成。

当结果表格里出现三个 `💾 已下载` 的状态时，说明整条链路是通的。

---

## 第五步：运行完整批量

点击 **▶ 开始完整批量**。

实时观察结果表格逐行更新。每条镜头会经历以下状态：

| 状态 | 含义 |
|------|------|
| ⏳ 等待中 | 还没提交，排队等候 |
| 📤 已提交 | 任务已提交，在服务端队列中等待 |
| 🎬 生成中 | 正在渲染中 |
| 💾 已下载 | 完成，MP4 文件已保存到本地 |
| ❌ 失败 | 生成失败，查看表格中的错误码和描述 |

15 条镜头在默认并发数（5）下，通常 **5–15 分钟**完成。

---

## 视频保存在哪里？

输出文件保存在以下目录：

```
output/
└── batch_20260423_143022/       ← 批量任务开始时间戳
    ├── ep01_s01_001.mp4
    ├── ep01_s01_002.mp4
    ├── ep01_s01_003.mp4
    ├── ...
    └── result_manifest.csv      ← 完整运行记录
```

每次批量运行会创建一个带时间戳的文件夹，里面是以 `shot_id` 命名的 MP4 文件，以及一份 `result_manifest.csv` 任务记录。

---

## 第六步：处理失败的镜头

如果有行显示 `❌ 失败`：

1. 查看结果表格中的 **错误码** 和 **详细说明** 列
2. 按下方表格找到原因并修复
3. 点击 **🔁 仅重试失败项** — 只会重跑失败的行，成功的行不会重复计费

| 错误 | 可能原因 | 修复方式 |
|------|----------|---------|
| `401 未授权` | 密钥错误或已过期 | 在控制台重新生成密钥 |
| `402 余额不足` | 积分用完了 | 控制台 → 计费 → 充值 |
| `429 请求太频繁` | 并发数过高 | 把 **并行镜头数** 调低至 2–3 |
| 参考图找不到 | 文件名不匹配 | 在侧边栏上传对应图片，或在 CSV 里换成 `https://` 链接 |
| `content_policy.pre` | 提示词被内容审核拦截 | 调整提示词措辞 |

```bash
# 启动 Batch Studio
cd toolkit/batch_studio && streamlit run app.py

# 用环境变量避免每次重启都要重新粘贴密钥
export NEXTAPI_KEY=sk_live_yourkey
export NEXTAPI_BASE_URL=https://api.nextapi.top
streamlit run app.py
```
