# 常见问题

---

## 基础问题

### NextAPI 是什么？

NextAPI 是基于 Seedance 搭建的视频生成网关，负责处理认证、计费、限速和参考图管理，让你专注于生成内容，不用操心底层基础设施。

### 可以生成什么类型的视频？

竖屏（9:16）、横屏（16:9）和方形（1:1）视频，时长 2–12 秒。主要使用场景：
- 短剧和剧集批量制作
- 电商产品展示
- 社交媒体内容规模化生产

### 我需要自己的 GPU 吗？

不需要。生成在远端基础设施上运行，你只需要网络连接和 API 密钥。

### 提示词支持中文吗？

英文提示词（`prompt_en`）效果最好，输出最稳定。中文提示词可以通过清单里的 `prompt_cn` 字段传入，当 `prompt_en` 为空时作为备用。

## 计费与积分

### 积分是怎么算的？

每条视频的消耗根据时长和复杂度计算。一条标准的 5 秒镜头通常消耗 **1–2 积分**。提交时 API 会返回 `estimated_credits`，完成后按实际消耗结算。

### 任务失败了还收费吗？

不收。失败任务的预占积分全额退回。

### 积分会过期吗？

与组织账户绑定，正常情况下不会过期。具体条款请参考控制台里的计费说明。

### 能免费测试吗？

Batch Studio 的 **快速测试（3 条镜头）** 功能只提交前 3 行，3–6 积分就能跑完，主要用于验证连接和画质，不是免费生成。

## 生成质量

### 角色跨镜头的一致性有多高？

提供高质量参考图、在提示词中固定角色特征，并设置 `continuity_group` 的情况下，同一场景序列内的角色一致性**非常高**。但在不同拍摄时段或复杂多变的场景间，可能会有轻微差异。

:::caution 一致性高，但不完美
模型不像真实演员一样记住角色身份。每条提示词里都要带上角色的固定特征描述（发型、标志性五官细节等），不能只靠参考图。
:::

### 为什么某些镜头里角色差别很大？

常见原因：

1. **提示词不一致** — 不同行的角色描述出现了偏差（比如发型描述不同）
2. **参考图质量差** — 模糊、裁剪过小、强背光的参考图效果很弱
3. **文字和图片描述冲突** — 提示词说"短发"但参考图是长发
4. **没有设置连贯组** — 应该共享上下文的镜头被分散到了不同组或根本没有组

修复方式：统一清单中的角色特征文字，建立 `character_bible.csv` 作为唯一来源。

### 参考图的格式要求是什么？

- **格式：** JPG 或 PNG
- **尺寸：** 最小 512×512，推荐 1024×1024 及以上
- **内容：** 正面、光线均匀、无人群、无文字水印
- **服装：** 需要特定造型一致性时，单独准备 `outfit_ref`，不要只靠角色图兼顾

### 为什么输出视频有些模糊或质量偏低？

- 提示词不够具体——越模糊的描述越容易出模糊的画面
- 时长设太短（2–3 秒）配合复杂动作，模型没有足够时间建立场景
- `negative_prompt` 里没有加 `low quality, blur`

## Batch Studio

### Batch Studio 可以跑在服务器或 CI 上吗？

可以。它是一个 Streamlit 应用，Python 能跑的地方都能运行。无界面使用时，可以直接调用 `api_client.py` 和 `batch_runner.py`，不依赖 Streamlit UI。

### API 密钥存储在哪里？

只存储在浏览器的会话状态中，不写入磁盘，除了 `api.nextapi.top` 不会发送到任何其他地方。

### 批次进行中关掉浏览器标签会怎样？

批次会停止。已经提交的任务会在服务端继续渲染，如果你知道 job_id 还可以手动获取结果。待提交的行就不会再发了。

### 能不能暂停批次，之后再继续？

当前版本不支持。请规划好一次性跑完。对于超过 200 条的大批次，建议按 50–100 条拆分成多批次运行。

### 我的 CSV 有 500 行，Batch Studio 能处理吗？

能。清单大小没有硬性限制。对于大批次，建议并发数设 5–8，预计需要 1–3 小时。保持电脑不睡眠、浏览器标签页保持打开。

## ComfyUI

### ComfyUI 节点可以离线使用吗？

不行。每个节点调用都需要向 NextAPI 服务器发起 HTTP 请求，需要网络连接。

### 可以在云端 ComfyUI 实例中使用这些节点吗？

可以——在云实例的环境里设置 `NEXTAPI_KEY` 环境变量，或者直接填在 Auth 节点里。使用云实例时要注意，不要把含有密钥的工作流 JSON 文件上传到公开位置。

### 一个 ComfyUI 工作流能跑多少条镜头？

ComfyUI 默认串行执行队列（一次一个）。超过 5 条镜头以上，建议改用 Batch Studio 的异步并发模式，效率明显更高。

## API

### 支持 Webhook 通知任务完成吗？

当前公开 API 不支持。请轮询 `GET /v1/jobs/{id}` 直到状态变为 `succeeded` 或 `failed`。推荐轮询间隔为 4 秒。

### 可以取消正在运行的任务吗？

暂不支持取消。任务一旦提交就会运行到完成或失败。服务商原因导致的取消，积分会自动退回。

### 视频链接有效期多长？

`video_url` 是签名链接，生成后 **24 小时**内有效。请及时下载并保存到自己的存储服务，不要依赖链接长期有效。

### 有沙箱或 Mock 模式吗？

开发和集成测试时，可以把 `NEXTAPI_BASE_URL` 指向本地 Mock 服务器。`toolkit/batch_studio/sample_data/README.md` 里有运行轻量级 Python Mock 服务器的说明，会返回假的任务 ID 和 MP4 链接，方便调试。
