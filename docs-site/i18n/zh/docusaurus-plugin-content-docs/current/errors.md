# 错误与排查

出现错误时，API 会返回 HTTP 状态码和如下格式的 JSON 响应体：

```json
{
  "error": {
    "code": "insufficient_balance",
    "message": "Organisation has no remaining credits."
  }
}
```

在 Batch Studio 中，错误码和错误说明会直接显示在结果表格的对应行里。

---

## 401 — 未授权

**发生了什么：** API 密钥未被识别，或者请求里根本没带密钥。

**常见原因：**

- 粘贴时有前后多余的空格或字符缺失
- 密钥在控制台已被撤销
- 用了测试密钥调用生产接口，或反过来
- 请求头写成了 `API-Key` 或 `X-Api-Key`，不是标准的 `Authorization: Bearer`

**修复方式：**

1. 打开控制台 → **密钥**，确认密钥存在且未撤销
2. 重新复制密钥，先粘贴到纯文本编辑器里检查有没有不可见字符
3. 确认请求头格式为 `Authorization: Bearer sk_live_…`

```bash
# 直接测一下你的密钥
curl https://api.nextapi.top/v1/videos/does-not-exist \
  -H "Authorization: Bearer sk_live_yourkey"
# 应该返回 404，而不是 401
```

---

## 402 — 余额不足

**发生了什么：** 组织账户的积分已用完。

**修复方式：** 控制台 → **计费 → 充值**。充值后立即生效，重试请求即可成功。

:::info 积分是组织共享的
同一个组织下的所有密钥共用一个积分余额。某个成员跑了大批量任务，会影响其他人的可用额度。
:::

---

## 429 — 请求太频繁

**发生了什么：** 超过了你的密钥每分钟请求限制（RPM）。

**即时处理：** API 会自动退避重试 429 错误——如果你用的是 Batch Studio 或 toolkit 客户端，等一等就会自动恢复，不需要手动干预。

**如果一直出现 429：**

- 在 Batch Studio 里把 **并行镜头数** 调低（比如从 10 降到 3）
- 控制台 → **密钥 → 编辑**，提高该密钥的 `rate_limit_rpm` 上限

**退避策略示意：**

```
第 1 次: 429 → 等 1.5s → 重试
第 2 次: 429 → 等 3s   → 重试
第 3 次: 429 → 等 6s   → 重试
第 4 次: 429 → 等 12s  → 重试
第 5 次: 失败 — 该行标记为 failed，错误码 http_429
```

用 **重试失败项** 在稍等一会儿后继续。

---

## 400 — 请求格式错误

**发生了什么：** 请求体有字段缺失或字段值不合法。

**常见原因和修复：**

| 错误提示 | 修复方式 |
|---------|---------|
| `"prompt is required"` | 对 `POST /v1/videos` 需在 `input` 中提供 `prompt`；Batch Studio 清单里加非空 `prompt_en`（或模板要求的那一列） |
| `"duration out of range"` / `duration_seconds` | 主接口使用 `input.duration_seconds`；旧版平铺体使用 `duration_seconds`；有值时一般为 **2–15** 秒 |
| `"unsupported aspect_ratio"` | 使用：`16:9`、`9:16`、`1:1`、`4:3`、`3:4`、`21:9`、`adaptive` |
| `"invalid reference url"` | 参考图必须是 `https://` 开头且可公开访问的链接 |

提交批量任务之前，先点 **🔍 验证 CSV**——大部分 400 类错误在验证阶段就能发现，不会消耗积分。

---

## 400 — content_policy.pre

**发生了什么：** 提示词在提交前被内容审核过滤器拦截，任务未创建。

这是硬拒绝——**不扣积分**。

**修复方式：** 修改提示词。避免：
- 暴力或露骨描述
- 公众人物姓名出现在不当语境中
- 敏感政治或宗教内容

如果你认为拦截有误，尝试用更中性的语言重写。具体阈值取决于你的组织在 控制台 → **内容审核** 里的配置。

---

## 5xx — 服务端错误

**发生了什么：** NextAPI 或视频生成服务侧出现异常。

**修复方式：**

1. 等 30–60 秒后重试
2. 如果持续失败，查看 NextAPI 状态页
3. Batch Studio 和 toolkit 客户端会自动退避重试

5xx 错误不扣积分。所有重试都失败时，预占积分会退回。

---

## 任务状态 `failed`（任务已创建后失败）

任务已成功创建并开始渲染，但在生成过程中失败了。

查看 `GET /v1/videos/{id}` 的 `error_code` 和 `error_message`（或旧版 `GET /v1/jobs/{id}` 若你仍用老创建端点）：

| error_code | 含义 | 修复方式 |
|------------|------|---------|
| `content_policy.post` | 生成的内容被事后审核拦截 | 软化提示词；检查内容审核配置 |
| `provider_error` | Seedance / 模型侧返回错误 | 重试，通常是偶发性问题 |
| `timeout` | 超过 15 分钟生成窗口 | 服务商拥堵，错峰重试 |
| `quota_exceeded` | 该组织的服务商配额已满 | 联系支持提升配额 |

## 任务超时

**发生了什么：** Batch Studio 的最大轮询时间（15 分钟）到了，任务还没返回终态。

说明服务商接受了任务但没能在时限内完成。

**修复方式：** 点击 **🔁 仅重试失败项**，创建一个新任务重新提交。旧任务最终可能会完成，但结果不会被下载。

如果超时频繁发生，查看 NextAPI 状态页，了解服务商队列的当前情况。

---

## 参考图找不到

**发生了什么（Batch Studio 中）：** 清单里的 `character_ref`、`outfit_ref` 或 `scene_ref` 找不到对应上传的文件。

**验证警告提示：**
> `'char_lin_ref.jpg' 已填写为角色参考图，但侧边栏中没有找到匹配的上传文件`

**修复方式：**

1. 在 Batch Studio 侧边栏上传文件——文件名必须与 CSV 里的**完全一致**（包括大小写和扩展名，例如 `char_lin.jpg` ≠ `Char_Lin.JPG`）
2. 或者把 CSV 里的值改成可公开访问的 `https://` 链接
3. 或者如果这条镜头不需要参考图，把对应字段留空

## 通用排查清单

联系支持之前，先检查这几项：

- [ ] API 密钥以 `sk_live_` 开头？
- [ ] 密钥前后没有多余空格？
- [ ] 控制台里密钥是激活状态（未撤销）？
- [ ] 组织积分余额大于零？
- [ ] `duration` 在 2–12 之间？
- [ ] `aspect_ratio` 是支持的值？
- [ ] 参考图是 `https://` 链接且可公开访问？
- [ ] 提示词不为空且有足够的描述细节？
- [ ] 提交前点了**验证 CSV**？

以上都检查过还有问题，提交工单时请提供：

- `job_id`（如果创建失败则提供完整的 API 响应）
- `error_code` 和 `error_message`
- 使用的提示词（如涉及保密内容可部分脱敏）
